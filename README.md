This program is described in the following publication:</p>

<h1><a id="user-content-requirements" class="anchor" aria-hidden="true" href="#requirements"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Requirements</h1>
<ul>
<li>Linux-based command line (Terminal). If you have MacOS, you just need to open Terminal. If you have Windows 10, you can run Linux command line as well ‚Äì <a href="https://www.howtogeek.com/249966/how-to-install-and-use-the-linux-bash-shell-on-windows-10/" rel="nofollow">check out this useful article</a></li>

<h1><a id="user-content-sample-experimental-design-and-basic-run" class="anchor" aria-hidden="true" href="#sample-experimental-design-and-basic-run"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Sample Experimental Design and Basic Run</h1>
<h2><a id="user-content-input-files-and-experimental-design" class="anchor" aria-hidden="true" href="#input-files-and-experimental-design"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Input files and experimental design</h2>
<p>Suppose you are interested in finding the genomic location of insecticide (I) resistance loci. First, you individually cross five insects from a resistant population (I_R) with a sensitive population (I_S), and expand the resulting segregating populations. After several generations, subject a subset of each of the five replicates to the pesticide. The surviving mites become your selected populations (referred to as Selected_Offspring in the example below) while the total populations are the unselected (referred to as Control_Offspring in the example below). You then extract and sequence DNA from each selected and unselected population as well as from both parental strains (I_R and I_S). More information and suggestions on experimental design for BSA studies are available in the publication specified above.
After you sequence each sample and get the fastq files containing the sequence reads, you map each sample to the reference genome (recommend BWA; <a href="http://bio-bwa.sourceforge.net/" rel="nofollow">Burrows-Wheeler Aligner</a>), and predict variants (recommend using GATK; <a href="https://software.broadinstitute.org/gatk/" rel="nofollow">Genome Analysis Tool Kit</a>) to get a variant call format (VCF) file, which will be used as the input file for program.</p>
<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Usage</h2>
<p>You get your VCF file and you are ready to find your pesticide resistance locus. To run the basic command, use the following template:</p>
<pre><code>python /Users/Say_My_Name/My_BSA/bsa_code.py \
-v /Users/Say_My_Name/Variant_Calls/my_stuff.vcf \
-psel I_R \
-pcon I_S \
-osel Selected_Offspring_1,Selected_Offspring_2,Selected_Offspring_3,Selected_Offspring_4,Selected_Offspring_5 \
-ocon Control_Offspring_1,Control_Offspring_2,Control_Offspring_3,Control_Offspring_4,Control_Offspring_5 \
-o /Users/Say_My_Name/My_BSA/Outfiles
</code></pre>
<p>To read more about these options and check all of them, run:</p>
<pre><code>/Users/Say_My_Name/My_BSA/bsa_code.py ‚Äìh
</code></pre>
<p>The basic command is best suited for a situation in which both of your parental strains have been inbred and sequenced. This method only considers homozygous loci, which might limit your power. Please see the <a href="#Two-inbred-parental-strains-not-present">Two inbred parental strains not present</a> section to see options that deal with other scenarios.</p>
<p>You may notice that you can provide single parental strains and at the same time have multiple replicates. However, if you use multiple parental strains, they should also be separated by a comma and come in the same order as their respective offspring. Order matters!</p>
<p>This basic run will determine the I_R allele frequency difference between each pair of selected and control offspring groups in sliding windows of 75kb (default option) that move across the genome in increments of 5kb (default option). It will output a number of files and plots. Note that the window size is based on the two-spotted spider mite (<em>Tetranychus urticae</em>) genome (~90Mb in size) and will need to be changed if your genome size is different. For corn (Zea mays), we use a window size of 5Mb and a slide of 500kb (e.g., <code>‚Äìw 5000000 ‚Äìs 500000</code>).</p>
<p>The basic run will output the following directories (within the major output directory you define as <code>‚Äìo</code>):</p>
<p><code>/BSA_output</code> will have files from the sliding window analysis and statistics (see below).
<code>/BSA_plots</code> will by default have three plots:</p>
<p><code>/BSA_plots/BSA_average_plot.pdf</code> plots a single line that represents the average of all selected-control offspring pairings.</p>
<p><code>/BSA_plots/BSA_comb_plot.pdf</code> plots each selected control pairing individually.
<code>/BSA_plots/BSA_sep_plot.pdf</code> plots each selected and control sample separately in two different colors ‚Äì one for the selected offspring groups and one for the controls.</p>
<hr>
<h1><a id="user-content-statistics---is-it-likely-selection-or-drift" class="anchor" aria-hidden="true" href="#statistics---is-it-likely-selection-or-drift"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Statistics - Is it likely selection or drift?</h1>
<h2><a id="user-content-overview" class="anchor" aria-hidden="true" href="#overview"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Overview</h2>
<p>Our package performs a simulation (which is a type of permutation that shifts allele frequency data around without changing the order) that determines whether your BSA peaks likely arose due to selection or genetic drift (please see <a href="https://doi.org/10.1534/genetics.118.301803" rel="nofollow">Wybouw, Kosterlitz, et. al. 2019</a> for an explanation of this method). We use a false discovery rate (FDR) of 0.05 (default option), but it can also be defined by the user with the ‚Äìsig flag (e.g., <code>‚Äìsig 0.01</code> for FDR of 0.01). The simulation relies on the power of replication ‚Äì the more replicates you have, the better. It works by combining your selected samples with the respective control samples. If you used different parental strains in your BSA, you will need to run the code individually for each set of parents for this to work correctly. If you want to run simulations on your data, add the ‚Äìperm flag (e.g., <code>perm ‚Äìi 10000</code>) to your command above. This method assumes that your selected and control methods are paired. Refer to the section below (<a href="#treating-unpaired-data">Treating unpaired data</a>) if otherwise.</p>
<h2><a id="user-content-treating-unpaired-data" class="anchor" aria-hidden="true" href="#treating-unpaired-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Treating unpaired data</h2>
<p>It often happens that selected and control offspring are not paired and thus do not represent true replicates (please see the tomato selection experiment in <a href="https://doi.org/10.1534/genetics.118.301803" rel="nofollow">Wybouw, Kosterlitz, et. al. 2019,</a> for an example). Use <code>‚Äìu</code> for unpaired data. The order you put in is the order that will be used to pair them for plotting in <code>BSA_average_plot.pdf</code>. The crucial distinction is how permutations are performed. Because data are unpaired, every potential pairing has to be tested. Therefore, you may want to allow more time to process the permutations in this event, especially if you have many samples. By default, every sample combination will be permuted, which is the factorial of the number of samples. This means that if you have five replicates, <code>‚Äìu ‚Äìperm 10000</code> will perform a total of 1,200,000 permutations.
With more permutations, this process will get very computationally intensive. Multiprocessing is incorporated into the code, and by default, the program will use all available processing cores on your machine. You can also specify how many cores you want to use with the <code>‚Äìn</code> flag. To further reduce processing time, you can select how many random combinations you want to permute by using the <code>‚Äìcomb</code> flag. For example, <code>‚Äìu ‚Äìperm 10000 ‚Äìn 30 ‚Äìcomb 60</code> will randomly select 60 selected-control offspring group combinations, and permute each 10k times using 30 processing cores. If you have many replicates (e.g., 10), the number of combinations rises quickly (with 10 replicates, to over 3.5 million). It is thus highly advised that you choose the number of desired combinations, e.g., <code>‚Äìcomb 120</code>, if you have a large number of replicates.</p>
<hr>
<h1><a id="user-content-two-inbred-parental-strains-not-present" class="anchor" aria-hidden="true" href="#two-inbred-parental-strains-not-present"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Two inbred parental strains not present</h1>
<p>We recommend using inbred parental strains and sequencing them in the course of your experiment. However, we realize that this is not always possible. The methods described in this section might remedy your situation.</p>
<h2><a id="user-content-single-inbred-parental-strain" class="anchor" aria-hidden="true" href="#single-inbred-parental-strain"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Single inbred parental strain</h2>
<p>If you only have one of the two parental strains sequenced, use the <code>‚Äìpmaj</code> flag. The alleles coming from the unknown parental strain will be inferred. For instance:</p>
<pre><code>/Users/Say_My_Name/My_BSA/bsa_code.py \
 -v \/Users/Say_My_Name/Variant_Calls/my_stuff.vcf \
-pmaj I_R \
-osel Selected_Offspring_1,Selected_Offspring_2,Selected_Offspring_3,Selected_Offspring_4, Selected_Offspring_5 \
-ocon Control_Offspring_1,Control_Offspring_2,Control_Offspring_3, Control_Offspring_4, Control_Offspring_5 \
-o /Users/Say_My_Name/My_BSA/Outfiles
</code></pre>
<p>This method can also be used if one of your parents is heterozygous. Specify <code>‚Äìpmaj</code> for the inbred parent.</p>
<h2><a id="user-content-no-genetic-information-on-parental-strains" class="anchor" aria-hidden="true" href="#no-genetic-information-on-parental-strains"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>No genetic information on parental strains</h2>
<p>If you have no parental information, do not specify the parents, and the program will perform BSA based on the absolute allele frequency difference between the selected and the unselected offspring. Important: This method treats unpaired data as is does paired data, i.e., it will force the pairings depending on the order you provide.</p>
<hr>
<h1><a id="user-content-additional-information" class="anchor" aria-hidden="true" href="#additional-information"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Additional information</h1>
<h2><a id="user-content-troubleshooting" class="anchor" aria-hidden="true" href="#troubleshooting"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Troubleshooting</h2>
<p>In a sliding window analysis, the minimum number of SNPs that have to be in a window to be considered is by default set as window_size*0.00025. Given that the default window size is 75kb, at last 19 SNPs have to be present in the window for it to be considered. If your parental strains have few SNPs compared to the reference genome, this setting might be too stringent. Use the <code>‚Äìm</code> flag followed the minimum number of SNPs of your choice to change this parameter.</p>
<h2><a id="user-content-masking" class="anchor" aria-hidden="true" href="#masking"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Masking</h2>
<p>While the actual BSA peak should have relatively smooth rise and fall, regions of misassembly in the genome can produce sudden sharp peaks. If it is reasonable to assume such regions are in fact misassembled, we provide an option to mask the region. The masking file should be created in a text editor (TextWrangler, Visual Studio Code, Notepad++, etc. Do not use programs like Word as they add special characters). The file should have tab-separated columns with chromosome (or scaffold name), beginning, and end position (bp) of the region to mask. For instance,</p>
<pre><code>chromosome_1	145000	155000
chromosome_4	1230000	1256000
</code></pre>
<p>After making the file, you specify its location, e.g.,
<code>-mask /Users/Say_My_Name/My_BSA/masking_file.txt</code></p>
<h1><a id="user-content-plotting-parameters" class="anchor" aria-hidden="true" href="#plotting-parameters"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Plotting parameters</h1>
<h2><a id="user-content-tick-marks-and-spacing" class="anchor" aria-hidden="true" href="#tick-marks-and-spacing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Tick marks and spacing</h2>
<p>When the plot is automatically generated, the program attempts to space the tick marks on the x-axis based on the genome size, while the y-ticks are automatically spaced 0.1 apart. You can change these options. This can come particularly handy when you are zooming in on a specific section of the genome (below). You can customize your plot with the following options (defaults shown below):</p>
<pre><code>-xstep 0
-ystep 0.1
-xticks 17
-yticks 0
-xminor 5
-yminor 2
</code></pre>
<p>The ‚Äústep‚Äù flags control how far apart the ticks are placed. The <code>‚Äìxstep</code> flag is set to 0 because the default plotting process will instead look a reasonable way to space the ticks (while aiming to make the total number of specified ticks; here it is set to 17 by <code>‚Äìxticks 17</code>) on the x-axis. Since the x-axis is longer, it will also include minor ticks that appear five times as frequently on the plot as the major ticks. You can control all of these options. However, one important thing to remember is that <code>‚Äìxstep</code> and <code>‚Äìystep</code> will override <code>‚Äìxticks</code> and <code>‚Äìyticks</code>. In other words, the program won‚Äôt use <code>‚Äìxticks</code> or <code>‚Äìyticks</code> if you specify <code>‚Äìxstep</code> or <code>‚Äìystep</code>. This means that since <code>‚Äìystep</code> is provided by default, you should specify <code>‚Äìystep 0</code> if you want your <code>‚Äìyticks</code> option to be used. It is important to point out that the automatic plotting methods should not be expected to always produce the best possible results. It may be helpful to adjust options to generate what you think would be the best plot.</p>
<h2><a id="user-content-zoom" class="anchor" aria-hidden="true" href="#zoom"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Zoom</h2>
<p>Sometimes it may be handy to zoom in on particular parts of the genome. To do so, you can provide a zoom file where each tab-delimited line (see <a href="#masking">Masking</a> above) specified the region you want to zoom in on. Add <code>-z /Users/Say_My_Name/My_BSA/zoom_file.txt</code> to your command. Note that since the default tick placing is 0.1, you should specify another tick spacing pattern if you want this to work well. For example, <code>‚Äìystep 0 ‚Äìyticks 10</code>. We also recommend that you use round numbers in your zoom file to avoid long decimals show up in the labels.</p>
<h2><a id="user-content-colors" class="anchor" aria-hidden="true" href="#colors"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Colors</h2>
<p>The default color scheme is ‚Äújet‚Äù. For Matplotlib color schemes, refer to <a href="https://matplotlib.org/examples/color/colormaps_reference.html" rel="nofollow">https://matplotlib.org/examples/color/colormaps_reference.html</a>.</p>
<p>The program takes schemes by default, but if you specify <code>‚Äìcol</code> custom,
you can choose your own colors. The github page has a pdf file with all the available color names (which we adapted from this post on Stack Overflow
<a href="https://stackoverflow.com/questions/22408237/named-colors-in-matplotlib" rel="nofollow">https://stackoverflow.com/questions/22408237/named-colors-in-matplotlib</a>). Alternatively, you can use hexadecimal color designations.</p>
<p>If you want to choose your own color scheme, use e.g.,
<code>-col custom,red,blue,royalblue,orange,#00FF00,darkviolet,indianred</code></p>
<p><code>/BSA_plots/BSA_sep_plot.pdf</code> will have red for selected and blue for the unselected groups
<code>/BSA_plots/BSA_comb_plot.pdf</code> will use royalblue, orange, #00FF00 (which is hex for lime green), darkviolet, and indianred for your five replicates.</p>
<h1><a id="user-content-independent-plotting" class="anchor" aria-hidden="true" href="#independent-plotting"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Independent plotting</h1>
<p>The package allows users to plot BSA files of their choice and the statistical cutoffs associated with them. This can come in handy when you are analyzing three separate experiments with several replicates, yet want to produce a plot that summarizes all of your findings in the same place. This is accomplished by using <code>‚Äìplot</code> and <code>‚Äìpermplot</code>. You can list your BSA sliding window files using the former and permutation files using the latter. For instance:</p>
<pre><code>-plot /Users/Say_My_Name/My_BSA/Outfiles_experiment1/BSA_output/selected_average.txt, /Users/Say_My_Name/My_BSA/Outfiles_experiment2/BSA_output/selected_average.txt,
/Users/Say_My_Name/My_BSA/Outfiles_experiment3/BSA_output/selected_average.txt

-permplot
/Users/Say_My_Name/My_BSA/Outfiles_experiment1/BSA_output/permutations.txt, /Users/Say_My_Name/My_BSA/Outfiles_experiment2/BSA_output/permutations.txt,
/Users/Say_My_Name/My_BSA/Outfiles_experiment3/BSA_output/permutations.txt

-o /Users/Say_My_Name/My_BSA/New_outfiles
</code></pre>
<p>You do not need to input anything else to get the plot. However, and this is crucial ‚Äì you do need to copy/paste an info_files directory into <code>/Users/Say_My_Name/My_BSA/New_outfiles</code> from one of your BSA runs. The program will automatically look for <code>/Users/Say_My_Name/My_BSA/New_outfiles/info_files/chrom_file.txt</code> to shade the chromosomes on your plot.</p>
<p>The colors will work the same in the sense that <code>‚Äìcol</code> Accent will select that color scheme, while <code>‚Äìcol custom,green,blue,violet</code> will use the three colors for your samples.</p>
